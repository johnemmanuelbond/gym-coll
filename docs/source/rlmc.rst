rlmc package
============

This package implements Monte Carlo (MC) methods for reinforcement learning. Monte Carlo methods are model-free algorithms that learn value functions from complete episodes of experience without requiring knowledge of the environment's transition dynamics. Unlike :doc:`temporal difference <td>` methods, MC methods wait until the end of an episode to update value estimates.

**Core Concepts:**

Monte Carlo methods are based on the fundamental idea of estimating expected values by averaging sample returns. They directly implement the definition of value functions:

.. math::

    V(s) = \mathbb{E}[G_t | S_t = s]

where :math:`G_t` is the return (cumulative discounted reward) from time step :math:`t`:

.. math::

    G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \ldots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}

The basic MC update rule for state values is:

.. math::

    V(s) \leftarrow V(s) + \alpha[G_t - V(s)]

where :math:`\alpha` is the learning rate and :math:`G_t` is the observed return from state :math:`s`.

Monte Carlo methods differ in how they handle multiple visits to the same state within an episode:

- **First-visit MC**: Only updates the value for the first occurrence of each state in an episode
- **Every-visit MC**: Updates the value for every occurrence of each state in an episode

Both approaches converge to the true value function as the number of episodes approaches infinity, though every-visit MC may have higher variance.

**Policy Types:**

Monte Carlo methods can be categorized into two main approaches:

**On-policy methods** improve the same policy that is used to generate episodes:
- Learn about policy :math:`\pi` from experience generated by following :math:`\pi`
- Must maintain exploration to ensure all state-action pairs are visited
- Typically use soft policies (e.g., :math:`\epsilon`-greedy)

**Off-policy methods** learn about one policy while following another:
- Learn about target policy :math:`\pi` from experience generated by behavior policy :math:`b`
- Requires importance sampling to correct for the difference in policies
- Can learn optimal policy while following exploratory behavior policy

**Hyperparameters**

- :math:`\alpha` (alpha): Learning rate for incremental updates (if used instead of sample averaging)
- :math:`\gamma` (gamma): Discount factor determining the importance of future rewards  
- :math:`\epsilon` (epsilon): Exploration parameter for epsilon-greedy policies in on-policy methods

All algorithms in this package implement these core MC principles with different approaches to policy improvement and exploration strategies.
